💡 GCN的核心思想

从CNN到GCN的类比

卷积神经网络(CNN)在图像处理中非常成功，它通过在规则网格上进行卷积操作来提取特征。 GCN将这个思想扩展到不规则的图结构上。

🖼️ CNN (图像)

  • • 在规则网格上卷积
  • • 每个像素有固定数量的邻居
  • • 参数共享，平移不变性
  • • 局部感受野逐层扩大

🕸️ GCN (图)

  • • 在不规则图上卷积
  • • 每个节点的邻居数量不同
  • • 参数共享，排列不变性
  • • 信息在图上传播聚合

🔬 GCN的数学原理

基本公式

GCN层的前向传播

H(l+1) = σ(D̃-1/2ÃD̃-1/2H(l)W(l))
H(l) 第l层的节点特征矩阵
W(l) 第l层的权重矩阵
Ã 带自环的邻接矩阵 A + I
D̃ Ã的度矩阵
σ 激活函数(如ReLU)

公式推导与理解

步骤1：简单的邻居聚合

最简单的想法：直接聚合邻居特征

H' = AHW

问题：不同度数的节点聚合强度不同

步骤2：度归一化

使用度矩阵进行归一化

H' = D-1AHW

改进：每个节点的邻居贡献被平均化

步骤3：加入自环

节点也要保留自己的特征

H' = D̃-1ÃHW, 其中 Ã = A + I

改进：节点考虑自己和邻居的特征

步骤4：对称归一化

使用对称归一化获得更好的数值稳定性

H' = D̃-1/2ÃD̃-1/2HW

最终形式：GCN的标准公式

🎯 图卷积的直观理解

信息传播过程

单次图卷积操作包含三个步骤：

1
线性变换

H(l)W(l) - 对每个节点的特征进行线性变换

2
邻居聚合

D̃-1/2ÃD̃-1/2 - 聚合邻居和自身信息

3
非线性激活

σ(·) - 引入非线性，增强表达能力

类比理解

社交网络中的信息传播：

• 每个人(节点)有自己的观点(特征)

• 与朋友(邻居)交流，整合不同观点

• 形成新的理解(更新后的特征)

• 通过多轮交流，信息传播到更远的人

多层GCN的效果

第1层GCN

每个节点聚合直接邻居的信息，感受野为1跳

第2层GCN

感受野扩展到2跳，能够利用更远的邻居信息

第L层GCN

感受野为L跳，但可能出现过平滑问题

💻 GCN代码实现

PyTorch实现

import torch
import torch.nn as nn
import torch.nn.functional as F

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super(GCNLayer, self).__init__()
        self.linear = nn.Linear(in_features, out_features)
        
    def forward(self, x, adj):
        # x: [N, in_features] 节点特征
        # adj: [N, N] 归一化的邻接矩阵
        
        # 线性变换
        h = self.linear(x)  # [N, out_features]
        
        # 图卷积：聚合邻居信息
        output = torch.mm(adj, h)  # [N, out_features]
        
        return output

class GCN(nn.Module):
    def __init__(self, num_features, num_classes, hidden_dim=16):
        super(GCN, self).__init__()
        self.gc1 = GCNLayer(num_features, hidden_dim)
        self.gc2 = GCNLayer(hidden_dim, num_classes)
        self.dropout = nn.Dropout(0.5)
        
    def forward(self, x, adj):
        # 第一层GCN + ReLU + Dropout
        h = F.relu(self.gc1(x, adj))
        h = self.dropout(h)
        
        # 第二层GCN + LogSoftmax
        output = self.gc2(h, adj)
        return F.log_softmax(output, dim=1)

# 邻接矩阵预处理
def preprocess_adj(adj):
    """
    对邻接矩阵进行GCN预处理
    """
    # 添加自环
    adj_with_self_loops = adj + torch.eye(adj.size(0))
    
    # 计算度矩阵
    rowsum = adj_with_self_loops.sum(1)
    degree_mat_inv_sqrt = torch.diag(torch.pow(rowsum, -0.5))
    
    # 对称归一化
    adj_normalized = torch.mm(torch.mm(degree_mat_inv_sqrt, 
                                     adj_with_self_loops), 
                            degree_mat_inv_sqrt)
    
    return adj_normalized

训练过程

# 训练GCN模型
def train_gcn(model, features, adj, labels, train_mask, val_mask, epochs=200):
    optimizer = torch.optim.Adam(model.parameters(), lr=0.01, weight_decay=5e-4)
    
    for epoch in range(epochs):
        model.train()
        optimizer.zero_grad()
        
        # 前向传播
        output = model(features, adj)
        
        # 计算损失（只在训练集上）
        loss = F.nll_loss(output[train_mask], labels[train_mask])
        
        # 反向传播
        loss.backward()
        optimizer.step()
        
        # 验证
        if epoch % 20 == 0:
            model.eval()
            with torch.no_grad():
                val_output = model(features, adj)
                val_pred = val_output[val_mask].max(1)[1]
                val_acc = val_pred.eq(labels[val_mask]).float().mean()
                print(f'Epoch {epoch}: Loss={loss:.4f}, Val Acc={val_acc:.4f}')

# 使用示例
features = torch.randn(100, 32)  # 100个节点，32维特征
adj = torch.randint(0, 2, (100, 100)).float()  # 随机邻接矩阵
labels = torch.randint(0, 3, (100,))  # 3类分类

# 预处理邻接矩阵
adj_norm = preprocess_adj(adj)

# 创建模型
model = GCN(num_features=32, num_classes=3, hidden_dim=16)

# 创建训练/验证掩码
train_mask = torch.zeros(100, dtype=torch.bool)
train_mask[:60] = True
val_mask = torch.zeros(100, dtype=torch.bool)
val_mask[60:80] = True

# 训练模型
train_gcn(model, features, adj_norm, labels, train_mask, val_mask)

⚖️ GCN的优缺点分析

✅ 优点

  • 高效计算

    基于矩阵运算，可以高效并行计算

  • 参数共享

    所有节点共享权重，参数量少

  • 数学简洁

    公式简单优雅，易于理解和实现

  • 理论基础

    有坚实的光谱图理论基础

❌ 缺点

  • 过平滑问题

    深层网络容易使节点表示趋于相同

  • 全图训练

    需要整个图的邻接矩阵，内存消耗大

  • 归纳学习能力弱

    难以处理新加入的节点

  • 同质性假设

    假设相邻节点相似，不适用于异质图
